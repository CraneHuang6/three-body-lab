# 模拟器三体星柔和跟随镜头设计

## 目标

让模拟器模式中的镜头随三体星运动而缓慢移动，使三体星始终保持在画面中心范围内，而不是长期漂到边缘或画外。

## 非目标

- 不修改观演模式镜头。
- 不引入缩放、旋转或视差。
- 不把镜头做成强锁定或弹簧过冲效果。

## 方案

采用“屏幕空间缓动跟随”：

1. 在模拟器模式中维护一个独立的摄像头中心点，单位仍然是世界坐标。
2. 三体星存活时，摄像头目标位置始终为三体星当前位置。
3. 每帧使用缓动插值让摄像头中心缓慢逼近目标，而不是瞬移到三体星位置。
4. 所有屏幕坐标换算统一走“世界坐标减摄像头中心”的映射，这样尾迹、碰撞特效、标签和天体本体会一起移动。
5. 若三体星死亡或不存在，摄像头保持在上一位置，不跳回世界原点。

## 用户可见行为

- 三体星会稳定停留在画面中心附近。
- 镜头会随着三体星运动缓慢平移，移动速度明显慢于“三体星瞬移到正中心”的硬锁定方案。
- 恒星、尾迹和碰撞特效会相对三体星做自然位移，不会出现三体星在动但背景绘制仍固定世界原点的割裂感。

## 实现边界

### 坐标系统

- 保留当前 `WIDTH` / `HEIGHT` / `WORLD_SCALE` 常量。
- 将 `sx` / `sy` 改为接收摄像头中心偏移，核心映射变为：
  - `screenX = CENTER_X + (worldX - cameraX) * WORLD_SCALE`
  - `screenY = CENTER_Y - (worldY - cameraY) * WORLD_SCALE`

### 镜头状态

- 在 `SimulatorMode.jsx` 中新增 `cameraCenter` 状态。
- 在 `simTime` 播放过程中，基于当前 snapshot 中的三体星位置更新 `cameraCenter`。
- 参数重置、切换预设或随机生成时，摄像头重置到当前三体星位置，避免新场景开始时出现旧镜头残留。

### 跟随强度

- 采用固定缓动系数的线性插值。
- 第一版目标是“柔和跟随”，优先避免晕动和抖动，不提供 UI 滑杆。

## 风险

- 如果缓动系数过低，三体星仍然可能偏离中心过多。
- 如果缓动系数过高，视觉上会退化成硬锁定。
- 如果只改天体本体、不改尾迹与碰撞特效，画面会出现分层错位；因此所有绘制入口必须统一走相机偏移映射。

## 验证

- 新增纯逻辑测试，验证给定三体星目标位置时，摄像头中心会按缓动规则向目标逼近。
- 新增渲染侧测试，验证 `sx/sy` 在应用摄像头偏移后会把三体星映射到中心附近。
